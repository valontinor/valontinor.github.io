Python 2.7.x VS 3.x

发表于 2016-10-10   |   分类于 技术

许多初学者都会迷茫应该从哪个版本的Python开始学起呢，我的答案是：从你喜爱的教程中的版本开始，等你学成后在来研究不同版本之间的差异。

但是，如果你准备开始进行一个项目那应该选择哪一个版本呢？我只能说这个不存在“对”的版本或“错”的版本，因为大部分的库都支持Python 2.7.x和Python3.x版本。但是，花点时间去了解Python 2.7.x和Python 3.x这两个Python常见版本之间的差异还是有必要的，这样可以使你在某个版本下进行开发时可以避开一些常见的陷阱，或者需要移植项目时。

  • __future__模块

Python 3.x引入了一些与Python 2不兼容的关键字和特性，在Python 2中可以通过__future__模块导入这些新内容。如果你希望在Python 2环境下写的代码也可以在Python 3.x中运行，那么建议使用__future__模块。例如，如果希望在Python 2中拥有Python 3.x的整数除法行为，可以通过下面的语句导入相应的模块：

  • print变成函数（Print Is A Function）

虽然print是Python 3中一个很小的改动，且已经广为人知，但依然值得提一下：Python 2中的print语句被Python 3中的print()函数取代，这意味着在Python 3中必须用括号将需要输出的对象括起来。

在Python 2中使用额外的括号也是可以的，但反过来在Python 3中想以Python 2中的形式不带括号调用print函数时，则会抛出SyntaxError异常。、

  • 整数除法

由于人们常常会忽略Python 3在整数除法上的改动（写错了也不会触发SyntaxError），所以在移植代码或在Python 2中执行Python 3中的代码时，需要特别注意这个改动。

所以，我还是会在Python 3的脚本中尝试用float(3)/2或3/2.0代替3/2，以此来避免代码在Python 2环境下可能导致的错误（或与之相反，在Python 2脚本中用from __future__ import division来使用Python 3的除法。

在Python 3中将Python 2中的long类型重命名为int类型，即，在Python 3中只有一种内置整数类型：int，但它可以和Python 2中的long类型一样使用。

  • Unicode

Python 2有基于ASCII的str()类型，其可通过单独的unicode()函数转成unicode类型，但没有byte类型。而在Python 3中，终于有了Unicode（utf-8）字符串，以及两个字节类：bytes和bytearrays。

  • xrange

在Python 2中，经常会用xrange()创建一个可迭代对象，通常出现在“for循环”或“列表/集合/字典推导式”中。

这种行为与生成器非常相似（如“惰性求值”），但这里的xrange-iterable是无尽的，意味着可能在这个xrange上无限迭代。由于xrange的“惰性求值”特性，如果只需迭代一次（如for循环中），range()通常比xrange()快一些。不过不建议在多次迭代中使用range()，因为range()每次都会在内存中重新生成一个列表。

在Python 3中，range()的实现方式与xrange()函数相同，所以就不存在专用的xrange()函数（在Python 3中使用xrange()会除法NameError）。

另一个值得一提的是，在Python 3中，range有了一个新的__contains__方法，__conatins__方法可以有效的加快Python 3中整数和布尔类型的“查找”速度。

  • next()函数和.next()方法

由于经常会用到next()、.next()函数（方法），这个在Python 3中也做了一些改动：在Python 2中，函数形式和方法形式都可以使用，而在Python 3中，只能使用next()函数（试图调用.next()方法会触发AttributeError）。

  • for循环与全局命名空间泄漏

在Python 3中，for循环中的变量不再会泄漏到全局命名空间中了：”列表推导式不再支持[… for var in item2, item2, …]这样的语法，使用[… for var in (item1, item2, …)]代替。还要注意列表推导式有不同的语义：现在列表推导式更接近list()构造器中的生成器表达式这样的语法糖，特别要注意的是，循环控制变量不会再泄漏到循环周围的空间中了。“

  • input()输入

Python 3改进了input()函数，使该函数总是将用户的输入存储为str对象，这样在Python 2中为避免读取非字符串类型会发生的危险行为而不得不使用的raw_input()函数成为了历史。
